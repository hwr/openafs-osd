backup addhost

目的

テープ・コーディネーターの項目をバックアップ・データベースに追加します。

構文 

backup addhost -tapehost <tape machine name> [-portoffset <TC port offset>]
               [-localauth]  [-cell <cell name>]  [-help]

backup addh -t <tape machine name>  [-p <TC port offset>]
            [-l]  [-c <cell name>]  [-h]

構文

backup addhost コマンドは、バックアップ・データベースにテープ・コーディネーターの項目を作成します。次の項目が作成されます。

  • テープ・コーディネーター (butc) プロセスが実行されているテープ・コーディネーター・マシンのホスト名。 -tapehost 引き数で指定します。
  • テープ・コーディネーターのポート・オフセット番号。 -portoffset 引き数で指定します。テープ・コーディネーター・マシンの /usr/afs/backup/tapeconfig ファイルにもポート・オフセットの項目が必要です。この項目は、磁気テープ装置の場合には UNIX の装置名に、バックアップ・データ・ファイルの場合にはパス名にマッピングされます。

各テープ・コーディネーターには、固有のポート・オフセット番号が必要です。要求したポート・オフセット番号に対応する項目がバックアップ・データベースに存在していると、このコマンドは失敗します。既存のテープ・コーディネーターを表示するには、 backup listhosts コマンドを使用します。

オプション

-tapehost
バックアップ・データベースにテープ・コーディネーターの項目を作成するマシンの完全修飾されたホスト名を指定します。このマシンは、セルのネーミング・サービス (ドメイン・ネーム・サービスなど) か、このコマンドが発行されたマシン上のホスト・ファイル (/etc/hosts または同等のファイル) に、項目が必要になります。

-portoffset
テープ・コーディネーターのポート・オフセット番号を指定します。 0 から 58510 までの範囲の整数を指定するか、この引き数を省略して、デフォルト値の 0 (ゼロ) を使用します。これは、-tapehost 引き数で指定されたテープ・コーディネーター・マシンの /usr/afs/backup/tapeconfig ファイルに、同じ組み合わせのテープ・コーディネーターと磁気テープ装置またはファイルが記述され、そのポート・オフセット番号とこの値が一致しなければなりません。

-localauth
ローカルの /usr/afs/etc/KeyFile ファイルのキーを使用して、サーバー・チケットを構成します。相互認証の間、 backup コマンド・インタープリターは、バックアップ・サーバー、ボリューム・サーバーおよび VL サーバーにこのチケットを提供します。このフラグは、-cell 引き数と一緒に使用してはなりません。詳細については、backup の解説ページを参照してください。

-cell
コマンドを実行するセルを指定します。この引き数は、 -localauth フラグと一緒に使用してはなりません。詳細については、backup の解説ページを参照してください。

-help
このコマンドのオンライン・ヘルプを出力します。これ以外の有効なオプションはすべて無視されます。

例

次のコマンドは、ポート・オフセット番号 4 を backup1.example.com で実行されているテープ・コーディネーターに割り当てる項目をバックアップ・データベースに作成しています。 

   % backup addhost -tapehost backup1.example.com -portoffset 4

次のコマンドは、ポート・オフセット番号 0 を backup3.example.com で実行されているテープ・コーディネーターに割り当てる項目をバックアップ・データベースに作成しています。 

   % backup addhost backup3.example.com

必要となる特権

コマンドの発行者は、バックアップ・サーバーが実行されているすべてのマシン上の /usr/afs/etc/UserList ファイルにリストされていなければなりません。また、-localauth フラグを使用する場合は、ローカル・スーパーユーザー root としてサーバー・マシンにログオンする必要があります。
