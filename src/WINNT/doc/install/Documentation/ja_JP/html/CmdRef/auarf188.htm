kas interactive

目的

対話モードに切り替えます。

構文 

kas interactive [-admin_username <admin principal to use for authentication >]
                [-password_for_admin <admin password>]  [-cell <cell name>]
                [-servers <explicit list of authentication servers>+]
                [-noauth]  [-help]

kas i [-a <admin principal to use for authentication >]
      [-p <admin password>]  [-c <cell name>]
      [-s <explicit list of authentication servers>+]  [-n]  [-h]

構文

kas interactive コマンドは、コマンドの発行者と対話式セッションを確立します。デフォルトでは、コマンド・インタープリターは、ローカルの /usr/vice/etc/ThisCell ファイルに指定されているセルで、ローカルの /usr/vice/etc/CellServDB ファイルに指定されているすべての認証サーバーに認証されてローカル・ファイル・システムにログオンしているユーザーと、認証接続を確立します。代替 ID セル名、または認証サーバーのリストを指定する場合には、 -admin_username、-cell、または -servers 引き数をそれぞれ指定してください。発行者または認証サーバーのチケット獲得サービスの最大チケット存続時間が短くない限り、対話モードの存続時間は、6 時間継続します。

kas interactive コマンド以外に、対話モードに入る方法が 2 つあります。

  1. シェル・プロンプトで操作コードを付けずに kas と入力します。必要であれば、-admin_username、 -password_for_admin、 -cell、および -servers 引き数の 1 つ以上を組み合わせて使用してください。
  2. 指示されたセルで、ローカルの /usr/vice/etc/CellServDB ファイルにリストされている認証サーバーと指定された ID で接続を確立する場合には、 kas コマンドの後に、ユーザー名、セル名を @ 記号で分けて入力してください (例 : kas admin@example.com)。必要であれば、-servers 引き数を指定して、該当するセルに所属する認証サーバー・マシンの代替リストを指定してください。

対話モードに入ると、次のようになります。

  • システム (シェル) プロンプトの代わりに、ka> プロンプトが表示されます。このプロンプトでは、操作コードを入力してください。(コマンド・セット名 kas は不要です。)
  • コマンド・インタープリターは、発行者のパスワードを要求するプロンプトを表示しません。

    対話モードに入るときに指定した発行者の ID およびパスワード、関連するセル、および一連の認証サーバー・マシンは、このセッションで発行されるすべてのコマンドに適用されます。 (kas) noauthentication コマンドを使用しない限り、これらの情報はセッションが終了するまで変更できません。このコマンドを使用すると、現在の認証接続を非認証接続に変更することができます。対話モード時に発行されたコマンドで、 -admin_username、 -password_for_admin、 -cell、および -servers 引き数が使用された場合には、これらの引き数は無視されます。

認証サーバーと非認証接続を確立する場合には、 -noauth フラグを使用するか、または間違ったパスワードを指定してください。ただし、各認証サーバー・マシンで許可検査が使用不可にならない限りは、このようなセッションで特権操作を実行することはできません。

現在の認証接続を終了し、非認証接続を確立するには、 (kas) noauthentication コマンドを発行してください。対話モードのまま通常のシェル・プロンプトに戻るには、 (kas) quit コマンドを発行してください。

オプション

-admin_username
コマンド実行時の認証サーバーによる認証で使用されるユーザー ID を指定します。詳細については、kas の解説ページを参照してください。

-password_for_admin
コマンド発行者のパスワードを指定します。省略すると (推奨)、 kas コマンド・インタープリターは、プロンプトを出して、パスワードの入力を要求します。入力したパスワードは表示されません。詳細については、kas の解説ページを参照してください。

-cell
コマンドの実行を行うセルに名前を付けます。詳細については、kas の解説ページを参照してください。

-servers
接続確立時に使用させる認証サーバーを実行しているマシンの名前を指定します。詳細については、kas の解説ページを参照してください。

-noauth
コマンドの発行者に非特権 ID anonymous を割り当てます。詳細については、kas の解説ページを参照してください。

-help
このコマンドのオンライン・ヘルプを出力します。これ以外の有効なオプションはすべて無視されます。

例

以下の例は、特権ユーザー admin として対話モードに入ったユーザーを表しています。 

   % kas interactive admin
   Password for admin: admin_password
   ka>

必要となる特権

なし。
